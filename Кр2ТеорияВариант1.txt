1. ООП(Объектно-ориентированное программирование) - это одна из парадигм программирования(парадигма - набор правил, которые соблюдает 
программист-разработчик при написании кода. Суть ООП в том, что все его программы состоит из объектов. Каждый объект - это определенная сущность 
со своими полями и методами.
2. Основные принципы ООП:
Абстракция - характеристика сущности, которая отличает от другиъ сущностей
Инкаплуляция - скрытие реализации объекта(атрибутов и тел его методов)
Наследование - принцип, в соответсвие с которым данные родительского класса разрешается применять в дочернем
Полиморфизм - свойство элементов модели с одинаковыми именами иметь различное поведение
3. Если говорить об инкапсуляции более подробно, то этот принцип предполагает сокрытие внутренних деталей объекта от внешнего мира и 
предоставление только определенных методов для взаимодействия с ним.
как преимущество инкапсуляции можно выделить защиту данных, то есть определение доступа к полям объекта позволяет предотвращать нежелательные 
изменения данных и обеспечивать безопасность.
4. Класс в Java является основной концепцией ООП и представляет собой шаблон или чертеж для создания объектов. Он определяет поля и
 методы объекта. Когда объект создается на основе класса, он получает свою уникальную идентичность, но наследует все свойства и методы, 
определенные в классе.
Скруктура класса:
public class MyClass {
    //Поля класса
    private int myField;
    private String name;
    //Метод класса
    public void myMethod() {
        // Логика метода
    }
}
Создание объекта в Мейне: 
MyClass obj = new MyClass();
Здесь MyClass является типом данных, а obj - экземпляр (объект), созданный на основе данного класса с использованием ключевого слова new.
5. Объект и класс имеют разные роли в разработки программы.
Класс можно представить как чертеж или шаблон, который определяет поля и методы, которые будут унаследованы 
экземплярами этого класса. Объект, как раз таки, является этим экземпляром. Когда класс используется для создания объекта, объект получает 
уникальную идентичность, а также наследует свойства и методы, определенные в класс
Класс служит для создания объектов.  Объект представляет конкретную реальную сущность в программе. Объект имеет состояние в виде значений 
своих полей и поведение в виде своих методов.
